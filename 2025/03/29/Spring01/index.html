Spring框架的产生及Spring Boot的推出

Spring框架的产生

由于JavaEE框架的复杂性（比如假定的分布式系统），Spring是为了解决应用复杂性而产生的框架，它提供两种机制：控制反转（IoC）和面向切面（AOP），来提供任意的服务和增强Bean的任意特性，使得Spring本身容易掌握，又可以通过Bean管理来无限扩展功能。

注：Bean 是java中的类，通常用于封装数据、业务逻辑或服务。

Spring的缺点

  • 使用门槛高，入门Spring需很长时间
  • 对过时技术兼容，导致使用复杂度高（比如处理事务使用XML还是注解@Trasaction）
    Trasaction：Deposit(/dɪˈpɒzɪt/) money into or withdraw money from a bank account.
  • XML配置已不再流行
  • 集成第三方工具时，程序员还要考虑工具之间的兼容性
  • 系统启动慢，不具备热部署功能，完全依赖虚拟机或Web服务器的热部署

SpringBoot的推出

基于Spring的缺点，Spring开发人员推出SpringBoot，解决了以上缺点，SpringBoot能激素开发Web系统，更容易架构大的分布式系统

SpringBoot管理Spring容器、第三方插件，并提供很多默认系统级的服务，无论是简单的Web系统还是负复杂的系统，都只需要少量配置和代码就能完成。

名称 作用
spring-boot-starter 核心 Starter，包括自动配置支持、日志库和 YAML
spring-boot-starter-web 用于构建 Web 应用程序，包括 Spring MVC 和 Tomcat

spring-boot-starter-security 提供 Spring Security 支持，用于添加安全和身份验证
spring-boot-starter-test 提供用于测试 Spring Boot 应用的库，如 JUnit、Hamcrest 和 Mockito
spring-boot-starter-jdbc 用于支持 JDBC 数据库访问和数据库池配置
spring-boot-starter-cache 提供缓存支持，例如 EhCache、Caffeine 或简单的内存基缓存
spring-boot-starter-actuator 提供生产级别的应用监控和管理功能
spring-boot-starter-mail 用于支持发送电子邮件
spring-boot-starter-aop 包括 spring-aop 和 AspectJ，支持面向切面编程
spring-boot-starter-batch 用于支持 Spring Batch，一个批量处理框架
spring-boot-starter-data-redis 用于 Spring Data Redis 和 Lettuce 客户端的 Redis 键值数据存储
spring-boot-starter-data-mongodb 用于 MongoDB 文档数据库和 Spring Data MongoDB
spring-boot-starter-data-elasticsearch 用于 Elasticsearch 搜索和分析引擎以及 Spring Data Elasticsearch
spring-boot-starter-websocket 用于构建 WebSocket 应用程序
spring-boot-starter-thymeleaf 用于使用 Thymeleaf 视图构建 MVC Web 应用程序
spring-boot-starter-freemarker 用于使用 FreeMarker 视图构建 MVC Web 应用程序
spring-boot-starter-mustache 用于使用 Mustache 视图构建 MVC Web 应用程序

SpringBoot相对于Spring的优点

  • 实现约定大于配置，是一个低配置的应用系统框架，且能够使用Spring的大量功能
  • 提供内置的Tomcat或者Jetty容器
  • 通过依赖jar包管理、自动装配技术，容易支持与其他技术体系、工具集成
  • 支持热加载，开发体验好，也支持SpringBoot系统监控，方便了解系统运行状态
