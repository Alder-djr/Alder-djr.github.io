Java异常

认识异常

异常就是代表程序可能出现的问题，例如0不能作为除数等

以上代码运行会出现错误：

异常的体系

Java.lang.Throwable

  • Error：代表系统级别的错误（严重问题），不需要程序员使用
  • Exception：异常，代表程序可能出现的问题，程序员通常会使用Exception以及它的孩子来封装程序出现的问题
    • RuntimeException：及其子类都是指运行时出现的异常，编译阶段不会出现错误提醒，如数组索引越界异常
      • RuntimeException的子类
    • 其他异常：如编译时异常：编译阶段会出现的错误提醒，如日期解析异常

异常的作用

  • 异常是用来查寻Bug的关键参考信息
  • 异常可以作为方法内部的一种特殊返回值，以便通知上层调用者底层的执行情况

异常处理

常见处理方式：将异常抛给调用者，接着有两种方法

  • 调用者捕获异常，记录异常并响应合适的信息给用户，用户体验较好
  • 调用者捕获异常，尝试重新修复
